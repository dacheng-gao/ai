# 语言规则

## 目标

本规则用于约束 AI Agents 在工程协作中的语言选择，目标是同时满足以下要求：

- 面向中文用户与中文开发团队时，协作体验默认中文优先
- 面向代码资产与工程资产时，保持 English First，避免语言污染
- 面向终端用户时，默认提供中文文案，除非项目已明确要求多语言化
- 为 Codex、Claude 及类似 Agent 提供一致、可执行、可覆盖的 production 级规则

## 适用范围

本规则适用于以下内容：

- AI 对用户的回复
- AI 在 agentic coding 过程中的中间产物
- 设计文档、计划文档、评审文档、交接文档
- Git 协作文本
- 代码目录、文件名、代码、配置、接口定义
- 面向终端用户的默认文案

## 核心原则

### 1. Audience First

先看受众，再决定语言。

- 面向中文用户、中文协作方、中文评审方：默认中文
- 面向代码执行环境、程序接口、跨语言工程资产：默认英文
- 面向终端用户：默认中文，除非项目已明确启用多语言

### 2. English First for Code Assets

所有代码资产默认英文优先，避免中英文混杂导致的维护成本、兼容性问题和协作噪音。

### 3. Chinese First for Human Collaboration

所有面向中文用户和中文团队成员的协作内容，默认中文优先，确保沟通成本最低。

### 4. Explicit Override Wins

如存在显式用户要求、项目级规则或上层规范，按优先级覆盖本规则。

## 优先级

发生冲突时，按以下顺序裁决：

1. 用户当前回合的显式要求
2. 项目级规则，例如 `AGENTS.md`、`CLAUDE.md`、仓库内规则文件
3. 本全局语言规则
4. Agent 自身默认偏好

补充约束：

- 上层规则未明确覆盖时，不得绕过本规则自行切换语言
- 若两个规则冲突且无法安全推断，应优先保持代码资产英文、协作文案中文

## 强制规则

### AI 对话与协作文档

以下内容 MUST 默认使用中文：

- AI 对用户的回复
- 执行中的进度更新
- 设计文档
- 实施计划
- 任务拆解
- 代码评审意见
- 测试结论与验证摘要
- handoff / 交接文档
- issue 分析、故障分析、根因分析

补充要求：

- 文字应简洁、直接、可执行，避免中英混写式废话
- 专有名词、协议名、框架名、必要命令可保留英文原文
- 若用户明确要求英文输出，则本段默认规则失效

### Agentic Coding 中间产物

AI 在开发过程中生成的中间文档 SHOULD 以中文为主，包括但不限于：

- design doc
- implementation plan
- debugging notes
- review notes
- verification report
- release notes 草稿

但以下内容 MUST 保持英文或与工程现状一致：

- 代码块中的源码
- 命令行命令
- 配置 key
- API path
- schema 字段名
- 数据库表名、列名
- 文件路径

### 代码资产

以下内容 MUST 使用英文：

- 目录名
- 文件名
- 源码本身
- 类名、函数名、变量名、常量名、类型名
- package 名、module 名
- API endpoint
- URL path
- header key
- JSON / YAML / TOML / ENV 配置 key
- schema、DDL、migration 标识
- 日志事件名、指标名、追踪字段名
- 测试名中的代码标识部分

以下内容 SHOULD 使用英文：

- 代码注释中的技术说明，除非项目明确规定注释中文
- commit message
- branch name
- tag name
- PR title

以下内容 SHOULD 默认使用中文：

- PR description / PR body
- review discussion
- issue 描述
- 变更说明中的业务解释

### 终端用户文案

凡直接展示给终端用户的默认文案 MUST 使用中文，包括但不限于：

- 页面默认文案
- 按钮文本
- 表单提示
- 空状态
- toast / snackbar
- CLI 面向终端用户的帮助文本
- 默认错误提示
- onboarding 文案

限制条件：

- 除非项目已明确声明支持 i18n / l10n，否则不得先以英文作为默认用户文案占位
- 若项目已启用多语言，则默认中文应作为首个完整可用语言版本之一，而不是后补
- 面向国际开发者的开发者工具、SDK、开放平台文档不属于“终端用户文案”，可按项目策略单独处理

### 日志与错误信息

日志分两类处理：

- 面向开发者排障、机器采集、监控聚合的结构化日志字段 MUST 使用英文
- 面向中国终端用户直接展示的错误消息 MUST 默认使用中文

推荐模式：

- `message` 或用户可见字段：中文
- `event_name`、`error_code`、`trace_key`、结构化字段：英文

不得将整段中文自由文本塞入结构化 key 中。

## 术语规则

在中文语境下，以下内容 SHOULD 直接保留英文：

- 专有名词，例如 `React`、`Kubernetes`、`PostgreSQL`
- 标准协议与格式，例如 `HTTP`、`JSON Schema`、`OpenAPI`
- 行业通用术语，例如 `rollback`、`hotfix`、`schema`、`token`
- 翻译后会降低可读性或增加歧义的词

术语使用要求：

- 首次出现时，可采用“中文解释 + 英文原词”形式
- 后续出现时，优先使用团队最稳定、歧义最小的写法
- 不应为了“全中文”强行翻译成熟术语

## 多语言化规则

若项目未明确要求国际化，则默认视为“中文单语言优先”项目：

- 用户可见文案默认中文
- 不提前引入 i18n 框架
- 不为未来可能的多语言需求预埋复杂抽象

若项目明确要求多语言化，则：

- 用户可见文案不得硬编码在业务逻辑中
- 默认中文仍应是首批受支持语言之一
- key 命名使用英文
- locale 文案内容按各语言分别维护

## 禁止事项

以下行为 MUST NOT 出现：

- 在代码目录、文件名、标识符中使用中文命名，除非项目已有明确先例且必须兼容
- 在同一层级文档中无理由频繁切换中英语言
- 将本应英文的 key、path、schema、env 名称翻译成中文
- 将本应中文的终端用户默认文案先写成英文占位
- 为了形式统一，强行翻译成熟英文术语，导致表达更差
- 未经项目明确要求，在代码中预埋过度设计的多语言体系

## 例外场景

以下场景可按原始语言保留，不受“中文优先”约束：

- 第三方 API 契约
- 开源协议文本
- 法律、合规、许可证原文
- 外部系统返回值
- 标准错误码、标准字段名
- 社区约定俗成且英文更准确的接口与术语

若引用外部原文：

- 可保留英文原文
- 必要时补充中文解释
- 不应修改原始协议字段或标准命名

## Agent 执行指令

Agent 在执行任务时应按以下顺序决定语言：

1. 判断内容是否面向终端用户、开发协作方、还是代码执行环境
2. 判断是否存在用户显式要求或项目级覆盖
3. 若是协作内容，默认中文
4. 若是代码资产，默认英文
5. 若是终端用户内容，默认中文
6. 若项目已启用 i18n，则按项目语言架构执行

## 自检清单

交付前至少检查以下项目：

- AI 回复是否以中文为主
- 中间文档是否以中文为主
- 代码目录、文件名、标识符是否保持英文
- commit message / branch name / PR title 是否保持英文
- PR body / review discussion / issue 描述是否保持中文优先
- 终端用户默认文案是否为中文
- 结构化日志字段是否保持英文
- 是否错误引入了不必要的多语言抽象
- 是否存在无意义中英混写

## 一句话决策规则

默认策略可压缩为一句话：

面向人协作时中文优先，面向代码资产时英文优先，面向中国终端用户时默认中文，除非用户或项目规则明确覆盖。

## 示例附录

以下示例用于帮助 Agent 在高频场景下快速决策：

| 场景 | 默认语言 | 说明 |
|------|----------|------|
| AI 给用户的回复 | 中文 | 属于协作沟通 |
| design doc / plan / review notes | 中文 | 属于中间协作文档 |
| commit message | 英文 | 属于 Git 原子资产，便于统一规范 |
| branch name | 英文 | 属于工程资产 |
| PR title | 英文 | 便于和 commit、分支保持一致 |
| PR body | 中文 | 主要给中文协作者阅读 |
| issue 分析与排障结论 | 中文 | 属于团队协作内容 |
| 目录名、文件名、类名、函数名 | 英文 | 属于代码资产 |
| API path / config key / schema field | 英文 | 属于程序接口与结构化资产 |
| 页面按钮、表单提示、默认报错 | 中文 | 属于终端用户默认文案 |
| 结构化日志字段 | 英文 | 便于检索、聚合与跨系统集成 |
| 面向用户展示的错误消息 | 中文 | 终端用户可见内容默认中文 |

快速判断：

- 给人读、且读者主要是中文用户或中文协作者：优先中文
- 给程序读、给工具读、给跨系统集成读：优先英文
- 给中国终端用户直接看：默认中文
