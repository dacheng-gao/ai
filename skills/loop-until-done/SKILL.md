# 迭代直到完成

默认工作流：理解需求 → 计划 → 执行 → 验证 → 补缺迭代，直到满足需求。

## 标准流程
1. 从相关视角审视问题
2. 明确目标：写清成功标准与范围边界（可量化时量化）
3. 执行任务：按计划完成改动或分析
4. **产出物回读**：重新阅读自己生成的全部产出物
5. 按 Verification 门禁验证结果
6. 完成度检查（全部通过 → 停止；任一未通过 → 修复后重新检查，硬上限 3 轮）：

- 交付物已创建且可访问
- 对照原始请求逐条核对，每条标注 `Done` / `Partial` / `Skipped + 理由`
- 无 `Partial` 或未覆盖项
- 代码变更任务满足通用退出标准（含 diff 自审）
- 产出物通过 `code-quality.md` 五维度自检（无 Critical/Important 问题）
