# 回答问题

## 适用场景
- 解释代码、函数、架构
- 技术概念或最佳实践问答
- 对比方案 / 选型建议
- "这段代码做了什么"、"为什么这样写"

## 执行规则
1. 从相关视角审视问题
2. 问题含歧义时：列出可能的解读，请用户选择后再答
3. 直接给出答案，控制长度
4. 代码示例仅在必要时提供
5. 不确定时明确标注不确定的部分，禁止编造

## 退出标准
- 用户的每个子问题均已回答（逐条核对）
- 关键依据已给出（如有），且可验证
- 不确定的部分已显式标注
