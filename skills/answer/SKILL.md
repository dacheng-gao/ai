# 回答问题

## 适用场景
- 解释代码、函数、架构
- 技术概念或最佳实践问答
- 对比方案 / 选型建议
- "这段代码做了什么"、"为什么这样写"

## 约束
- 禁止执行变更操作（文件写入、git 变更、破坏性命令）。允许读取文件和执行只读命令以收集上下文

## 执行规则
1. 激活适用视角审视问题
2. 问题含歧义时：列出可能的解读，请用户选择后再答
3. 直接给出答案，控制长度
4. 代码示例仅在必要时提供
5. 不确定时明确标注不确定的部分，禁止编造

## 退出标准

| # | 标准 | 验证方式 |
|---|------|---------|
| 1 | 子问题均已回答 | 逐条核对用户问题 |
| 2 | 关键依据可验证 | 如有依据，已给出且可验证 |
| 3 | 不确定部分已标注 | 显式标注不确定的部分 |

> 质量门禁豁免：本技能为轻量问答，跳过代码质量检查。
